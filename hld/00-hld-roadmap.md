# HLD Roadmap

High Level Design is the skill of designing distributed systems that satisfy functional requirements, scale requirements, latency targets, availability needs, data consistency expectations, and operational constraints.

## Core Topics

### Foundations

- Client-server architecture
- HTTP/HTTPS
- DNS
- Load balancers
- Reverse proxies
- REST, GraphQL, gRPC
- Synchronous vs asynchronous communication

### Scalability

- Vertical scaling
- Horizontal scaling
- Stateless services
- Load balancing algorithms
- Partitioning
- Sharding
- Replication
- Caching
- CDN

### Data

- SQL vs NoSQL
- Indexes
- Transactions
- Replication
- Read replicas
- Sharding keys
- Denormalization
- Eventual consistency
- Search indexes
- Object storage

### Reliability

- Availability
- Fault tolerance
- Retries
- Timeouts
- Circuit breakers
- Bulkheads
- Backpressure
- Idempotency
- Disaster recovery

### Operations

- Metrics
- Logs
- Traces
- Alerts
- SLO/SLI/SLA
- Deployment strategies
- Blue-green/canary deploys

## Interview Progression

Beginner:

- URL shortener
- Rate limiter
- Pastebin
- Tiny Twitter feed

Intermediate:

- WhatsApp/Chat
- YouTube/Video streaming
- Instagram
- Notification system
- Food delivery

Advanced:

- Distributed cache
- Search autocomplete
- Payment system
- Ride sharing
- Distributed file storage
- Ticket booking

## What Interviewers Want

- Can you ask clarifying questions?
- Can you estimate scale?
- Can you choose storage correctly?
- Can you reason about consistency?
- Can you find bottlenecks?
- Can you explain tradeoffs?
- Can you design for failure?
